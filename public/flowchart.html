📊 Missingness Analysis Details

Rationale

Characterizing missingness patterns is critical because clinical data from retrospective cohorts often has informative missingness — the absence of a test result may itself carry diagnostic signal (e.g., karyotype ordered only when abnormality suspected).

Thresholds
  • 100% missing → automatic drop (no information)
  • >50% missing → flagged, retained if clinically meaningful
  • <50% missing → standard imputation eligible
Reference

Van Buuren, S. (2018). Flexible Imputation of Missing Data. CRC Press. — MCAR/MAR/MNAR framework applied.

Details

Baseline demographics (Age, Race) had 0% missing. Hormonal panel ranged from 22–42% missing. Comorbidities (Diabetes 99.1%, Hypertension 99.8%) were almost entirely unrecorded — likely documented only when present.

⚠️ High-Missingness Handling Strategy

Rationale

In clinical research, high missingness does not always justify exclusion. Karyotype and Y-microdeletion tests are expensive and typically ordered when clinical suspicion is high — making the missingness itself informative (MNAR pattern). Discarding these variables would lose valuable diagnostic signal.

Threshold

50% missingness threshold for flagging. Clinical domain knowledge used to override automated drop recommendations for diagnostically important variables.

Reference

Zeadna et al. (2020). Human Reproduction. — Retained genetic markers despite high missingness due to clinical informativeness.

Details
  • Retained examples: Karyotype (62.3%), Y microdeletion (62.4%), Testis size sono (82-83%)
  • Strategy: Binary missingness-indicator features created alongside original values
  • Validation: SHAP analysis later confirmed these missingness indicators carry predictive signal

STEP 07A — Paper 1

🔲 Missingness Indicators + SHAP

Binary features flag missing-vs-observed for each variable. SHAP validated that missingness itself carries clinical signal (e.g., karyotype not ordered = lower clinical suspicion).

Missingness = Signal SHAP-validated

🔲 Paper 1 Strategy

Rationale

In clinical settings, the decision to order a test is itself informative. A missing karyotype may indicate the clinician did not suspect a chromosomal abnormality. Creating binary missingness-indicator features captures this "decision-to-test" information that would be lost with standard imputation.

Threshold

Indicators created for all variables with non-trivial (>5%) missingness rates. Each indicator: 1 = missing, 0 = observed.

Reference

Jamalirad et al. (2026). Paper1_revised.md. — "Missingness-indicator features validated as informative by SHAP."

Details

Final feature set for Paper 1: 51 features including missingness indicators. SHAP global importance rankings showed several missingness flags among top-30 predictors.

STEP 07B — Paper B

🔄 kNN Imputation (Train Folds Only)

k-Nearest Neighbors imputation on continuous predictors. Fit on training folds only to prevent data leakage. Mode/explicit "Unknown" for categorical variables.

kNN imputation No leakage

🔄 Paper B Strategy

Rationale

kNN imputation leverages the local structure of the feature space — patients with similar clinical profiles inform missing values. This is preferable to mean/median imputation for clinical data where feature correlations are complex and non-linear.

Threshold

kNN fitted within CV training folds only; k selected via inner CV. Continuous predictors imputed after standardization scaffolding. Categorical variables: mode imputation or explicit 'Unknown' category.

Reference

Jamalirad et al. (2026). PaperB_revised.md. — "kNN imputation applied within CV folds to prevent leakage."
Troyanskaya et al. (2001). Missing value estimation methods for DNA microarrays. Bioinformatics.

Details

Final feature set for Paper B: 36 features after further pruning. All preprocessing (imputation + scaling + SMOTE) performed inside cross-validation loop.

🧬 NLP Pathology Parsing Details

Rationale

Histopathology reports are recorded as free text (e.g., "SCO (90%). CSTH (10%).") and cannot be directly used as ML features. Structured extraction converts qualitative descriptions into quantitative, per-side features while preserving the clinical hierarchy of maturation arrest stages.

Threshold
  • Regex extracts explicit % when present; assigns 100% if pattern found without percentage
  • MA hierarchy: Spermatogonial → Spermatocytic → ImmatureRound → Elongated
  • If arrested at earlier phase, later phases set to 0
Reference

Custom pipeline. Top pathology patterns: SCO (846 cases), SCO+CSTH (194), MA spermatocytic (120/115), CSTH (81), Hypospermatogenesis (38).

Details — 16 Extracted + 6 Aggregated

🔬 Feature Selection Pipeline Details

Rationale

No single feature selection method is universally optimal. Mann-Whitney U captures univariate distributional differences; Mutual Information captures non-linear dependencies; RFE-RF captures multivariate interaction effects. Using UNION (rather than intersection) ensures that features selected by any method are retained, reducing the risk of discarding informative variables.

Thresholds
  • Stage 1: Mann-Whitney U, p < 0.05 (univariate filter on continuous features)
  • Stage 2: Mutual Information ranking against Outcome1 (top features by MI score)
  • Stage 3: Recursive Feature Elimination with Random Forest (RFE-RF)
  • Final: UNION of all three → 51 features (Paper1), 36 features (PaperB)
Reference

Guyon, I. & Elisseeff, A. (2003). "An Introduction to Variable and Feature Selection." JMLR.
Ory et al. (2022). World Journal of Men's Health. — Used similar multi-stage feature selection for microTESE prediction.

STEP 12

📐 Standardization — Z-Score / RobustScaler

Continuous features standardized (mean=0, σ=1). Scaler fit on training set ONLY, then applied to test set. RobustScaler selected in best LightGBM pipeline.

Z-score (mean=0, σ=1) Train-only fitting RobustScaler (best pipeline)

📐 Standardization Details

Rationale

Different clinical variables have vastly different scales (e.g., Age in years vs. FSH in mIU/mL vs. testicular volume in mL). Without standardization, distance-based methods (kNN, SVM) and gradient-based optimizers are biased toward high-magnitude features. RobustScaler uses median/IQR, making it resistant to the outliers common in hormonal data.

Parameters
  • Z-score: μ=0, σ=1 for standard pipeline
  • RobustScaler: centering=median, scaling=IQR (selected by nested CV)
  • Scaler choice tuned inside cross-validation loop
Reference

Scikit-learn documentation. RobustScaler selected via RandomizedSearchCV (50 iterations, 2-outer × 5-inner CV).

Scope

Applied to: Age, Partner Age, BMI, Height, Weight, FSH, LH, Testosterone, Prolactin, E2, testicular volume measurements, Sakamoto volumes, Seminal plasma pH.

STEP 13

🔢 Categorical Encoding — One-Hot & Binary

One-hot encoding for multi-class categoricals (Race, Habits, Karyotype, Y-microdeletion, Occupation). Binary 0/1 for comorbidities and surgical history.

One-hot encoding Label encoding Binary flags

🔢 Categorical Encoding Details

Rationale

ML algorithms require numeric input. One-hot encoding preserves the nominal nature of multi-class variables without imposing ordinal assumptions. Binary indicators for comorbidities (Diabetes, Hypertension) and surgical flags are already naturally binary.

Details
  • One-hot: Race, Habits, Karyotype categories, AZFa/AZFb/AZFc, Occupation exposure
  • Binary (0/1): Diabetes, Hypertension, Varicocele, T.BX, Inguinal hernia, Orchiopexy, Surgery trauma(s)
  • Engineered binary: is_Primary, all pathology indicators (SCO, CSTH, MA stages, Hypospermatogenesis)
Reference

Standard practice. Bachelot et al. (2023). JMIR. — Similar encoding strategy for clinical prediction models.

⚖️ Class Imbalance Details

Rationale

With a 1.72:1 failure-to-success ratio, classifiers tend to be biased toward predicting failure. SMOTE generates synthetic minority (success) examples by interpolating between existing minority-class neighbors, improving the classifier's sensitivity without simply duplicating records.

Parameters
  • SMOTE applied inside CV training folds only (leakage prevention)
  • Never applied to test/validation sets
  • Original ratio: 1,527 failure : 886 success (1.72:1)
Reference

Chawla et al. (2002). "SMOTE: Synthetic Minority Over-sampling Technique." JAIR.
Xiao et al. (2025). Scientific Reports. — SMOTE in microTESE prediction context.

Rationale

Stratified splitting ensures that the class distribution (36.7% success) is preserved in both train and test sets. The 80/20 ratio balances having sufficient training data with a meaningful test set size. Fixed random_state ensures reproducibility.
