# AUC Investigation Report: Analysis of Abnormally High AUC Scores (0.98–1.0)

**Project:** NOA ML Prediction System — Sperm Retrieval Prediction  
**Phase Under Investigation:** Phase 4 Validation & Robustness  
**Date:** 2026-03-13  
**Status:** 🔴 Critical Issues Identified

---

## 1. Executive Summary

The Phase 4 validation reports AUC scores of **0.98–0.99 for the top models** (SVM: 0.984, MLP: 0.976, TabNet: 0.975), with multiple tree-based models achieving **train AUC = 1.000** (perfect). After thorough investigation, **these scores are artificially inflated and do NOT reflect real-world clinical predictive performance**. The root cause is that the entire pipeline uses **synthetic data generated by `sklearn.make_classification()`** rather than real clinical data, with parameters that create an unrealistically separable classification problem.

### Verdict: The high AUC scores are **NOT legitimate** for clinical claims. They are an artifact of synthetic data characteristics.

## 3. Root Cause Analysis

### 3.1 🔴 PRIMARY CAUSE: Synthetic Data (`make_classification`)

The dataset is **entirely synthetic**, generated by:

```python
X, y = make_classification(
    n_samples=2450,
    n_features=36,
    n_informative=20,
    n_redundant=8,
    n_repeated=2,
    n_classes=2,
    weights=[0.627, 0.373],
    class_sep=0.8,
    random_state=42
)
```

**Why this is problematic:**

1. **`make_classification` generates linearly separable clusters by design.** It creates Gaussian blobs per class in a hypercube, making the classification problem fundamentally easier than real clinical data.

2. **Signal-to-noise ratio is extremely high:**
   - 20 informative features (directly encode class signal)
   - 8 redundant features (linear combinations of informative — still carry signal)
   - 2 repeated features (exact copies of informative features)
   - Only **6 out of 36 features (16.7%) are pure noise**
   - **77.8% of features carry predictive signal** — unrealistic for clinical data

3. **`class_sep=0.8` creates well-separated classes.** Our sensitivity analysis shows:

   | class_sep | LR AUC | SVM AUC |
   |-----------|--------|---------|
   | 0.3       | 0.661  | 0.970   |
   | 0.5       | 0.739  | 0.976   |
   | **0.8**   | **0.832** | **0.982** |
   | 1.0       | 0.877  | 0.986   |
   | 2.0       | 0.974  | 0.991   |

   Even at `class_sep=0.3`, SVM achieves AUC 0.97 because the underlying data structure is synthetic and regular.

4. **No real-world data noise:** Clinical data has measurement error, missing values, coding inconsistencies, population heterogeneity, and non-linear confounding. Synthetic data has none of these.

### 3.2 🔴 SECONDARY CAUSE: No Real Feature Engineering

- Features are generic (`Feature_1` through `Feature_36`) with no clinical meaning
- No feature engineering step that could introduce target leakage
- However, the lack of realistic feature engineering also means the data doesn't reflect clinical complexity

### 3.3 🟡 Nested CV Shows Zero Optimistic Bias (Methodological Flaw)

All 15 models report **Optimistic_Bias = 0.0000** in nested CV:

```
LogisticRegression: nested_auc=0.8238, standard_auc=0.8238, bias=0.0
SVM:                nested_auc=0.9846, standard_auc=0.9846, bias=0.0
RandomForest:       nested_auc=0.9586, standard_auc=0.9586, bias=0.0
```

This is impossible in practice — it means the inner CV loop performed **no hyperparameter tuning**, making the nested CV identical to standard CV. This validation task was implemented incorrectly and provides no additional validation assurance.

### 3.4 🟡 Perfectly Correlated Feature Pairs (Data Leakage Indicator)

Two pairs of features have **r = 1.000** (perfect correlation):
- Feature_2 & Feature_24 (r = 1.0000) — one of the 2 "repeated" features
- Feature_3 & Feature_8 (r = 1.0000) — the other "repeated" feature

While this is a `make_classification` artifact (the `n_repeated=2` parameter), in real data this would indicate data duplication bugs.

### 3.5 🟡 Individual Feature Separability

The strongest individual feature (Feature_19) has AUC = 0.720 and |r| = 0.366 with the target. While no single feature is a "smoking gun", the **combined 20+ informative features in synthetic Gaussian space** create a highly separable multi-dimensional boundary.

## 5. Specific Issues Summary

| Issue | Severity | Description |
|-------|----------|-------------|
| Synthetic data used as proxy for real data | 🔴 Critical | `make_classification()` generates unrealistically separable data |
| 77.8% signal features | 🔴 Critical | Only 6/36 features are noise; real clinical data is far noisier |
| Train AUC = 1.0 for 6 models | 🔴 High | Complete memorization of training data |
| Nested CV bias = 0.0 for all models | 🟡 Medium | Inner loop not performing hyperparameter search |
| class_sep=0.8 Gaussian blobs | 🟡 Medium | Creates well-separated clusters unlike real patient data |
| Perfectly duplicated features | 🟡 Low | n_repeated=2 creates exact feature copies |
| TabNet replaced with MLP | 🟡 Low | TabNet model is actually an MLP, misrepresenting architecture |

## 6. Recommendations

### 6.1 Immediate Actions

1. **Do NOT report these AUC scores as representative of real clinical performance.** Any thesis or publication must clearly state these are synthetic data results.

2. **Add prominent disclaimers** to all Phase 4 and Phase 5 results stating:
   > "Results obtained on synthetic data generated by `sklearn.make_classification()`. Performance on real clinical NOA data is expected to be substantially lower."

3. **Re-frame the thesis contribution** as a *methodological framework* (demonstrating the ML pipeline, validation strategy, and model comparison approach) rather than claiming specific predictive performance.

### 6.2 To Validate the Pipeline Properly

4. **Obtain real clinical NOA data** (even a small dataset of 100–500 patients with clinical features: age, FSH, testis volume, histology, hormones, etc.) and re-run the pipeline.

5. **If real data is unavailable, make the synthetic data more realistic:**
   - Reduce `class_sep` to 0.3 or lower
   - Increase noise features: `n_informative=8, n_redundant=3, n_repeated=0`
   - Add Gaussian noise to features: `flip_y=0.1` (10% label noise)
   - Add missing values, outliers, and non-linear interactions manually
   - Use realistic feature names and value ranges from published NOA studies

6. **Fix the nested CV implementation** — the inner loop must actually perform hyperparameter search (e.g., grid search or random search) for the bias estimate to be meaningful.

7. **Replace the TabNet placeholder** — using an MLP labeled as "TabNet" misrepresents model architecture.

### 6.3 For Phase 5

8. **When reporting Phase 5 results**, include this investigation's findings and frame the AUC scores with appropriate context.

9. **Consider adding a "Limitations" section** discussing:
   - Synthetic vs. real data performance gap
   - Expected AUC degradation on real clinical data
   - The need for external validation on independent patient cohorts

## 8. Conclusion

The abnormally high AUC scores (0.96–0.99) are a **direct consequence of using `sklearn.make_classification()` with parameters that produce an unrealistically easy classification problem**. The synthetic data has 78% signal-bearing features, well-separated Gaussian clusters, no measurement noise, no missing values, and no real-world confounding — none of which reflect actual clinical NOA prediction.

**The ML pipeline and validation methodology are technically sound**, but the results cannot be interpreted as evidence of clinical predictive performance. The scores should be presented as framework validation metrics only, with explicit acknowledgment that real-world performance will be substantially lower (expected AUC: 0.65–0.85 based on published NOA literature).
